login(string  $username, string  $email, string|null  $password = null, array|null  $groups = null) : string

Logs the user in Flarum. Generally, you should use this method when an user successfully log into your SSO system (or main website). If user is already signed up in Flarum database (not signed up with this extension) you need to pass plain user password as third parameter (for example Flarum admin) You can also set groups to your users with an array

update()

update(string  $username, string  $email, string|null  $password = null) : mixed

Updates a user. Warning! User needs to be find with username or email, so one of those two has to be the old one

Parameters

string $username

Old/new username. Username will be changed if email matches the one in Flarum database, else it will be used to find the user ID
string $email

Old/new email. Email will be changed if username matches the one in Flarum database, else it will be used to find the user ID
string|null $password

New password (changes old password to this one)

getUsersList()

getUsersList(null|string  $filter = null) : array|\Illuminate\Support\Collection

Gets the list of the users actually signed up on Flarum, with all the properties

Parameters

null|string $filter

If set, returns the full users list (with other info) and not only the usernames Can be one of the following: type, id, attributes, attributes.username, attributes.displayName, attributes.avatarUrl, attributes.joinTime, attributes.discussionCount, attributes.commentCount, attributes.canEdit, attributes.canDelete, attributes.lastSeenAt, attributes.isEmailConfirmed, attributes.email, attributes.markedAllAsReadAt, attributes.unreadNotificationCount, attributes.newNotificationCount, attributes.preferences, attributes.canSuspend, attributes.bio, attributes.newFlagCount, attributes.canViewRankingPage, attributes.Points, attributes.canPermanentNicknameChange, attributes.canEditPolls, attributes.canStartPolls, attributes.canSelfEditPolls, attributes.canVotePolls, attributes.cover, attributes.cover_thumbnail, relationships, relationships.groups

There could be more if you have other extensions that adds them to Flarum API

array|\Illuminate\Support\Collection
